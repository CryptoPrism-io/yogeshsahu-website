Next.js vs Astro — For Your Portfolio

Pick the foundation that fits your ambition and workflow

B — Next.js + React
RECOMMENDED for your ambition level
What you get
  • ✓ Full React component model — reuse hero, nav, project cards
  • ✓ Framer Motion built in — scroll animations, page transitions, stagger effects
  • ✓ Canvas scroll-driver for Veo 3 frame sequence is trivial in a React useEffect
  • ✓ next/image — auto-optimises your extracted frames (WebP, lazy load)
  • ✓ App Router + Server Components — fast, SEO-friendly out of the box
  • ✓ Easy to add dynamic features later (blog via MDX, contact API route)
  • ~ Slightly heavier JS bundle than Astro (still fast with proper config)
  • ~ Needs Vercel or similar for best deployment experience
Best for: You want Framer Motion scroll effects, clean component architecture, and potential to grow (blog, API, dynamic data). This is the standard choice for ambitious dev portfolios in 2025.
STACK
Next.js 15 React 19 Framer Motion Tailwind CSS Canvas API Vercel
C — Astro
PERF-FIRST if bundle size is top priority
What you get
  • ✓ Ships zero JS by default — only interactive islands send JS
  • ✓ Near-perfect Lighthouse scores out of the box
  • ✓ Can use React, Vue, Svelte components side by side
  • ✓ MDX blog built in — great for writing
  • ~ Canvas scroll animation needs a client:load island — adds friction
  • ✗ No Server Actions / API routes as smooth as Next.js
  • ✗ Framer Motion works but feels bolted-on vs native React
  • ✗ Smaller community, fewer portfolio-specific tutorials
Best for: Mostly-static sites where you want maximum performance scores and don't need heavy interactivity. Great choice, but the scroll-driven Veo 3 animation + Framer Motion feel more natural in Next.js.
STACK
Astro 5 React islands Tailwind CSS Canvas API Netlify / Vercel
Verdict for YOUR specific build

You want scroll-driven Veo 3 frames + Framer Motion transitions + potential blog/contact API. Next.js wins here — the Canvas API + Framer Motion + React useEffect combo is battle-tested for exactly this kind of scroll animation. Astro would work but adds friction at every animated section. Both deploy to Vercel/Firebase for free.
