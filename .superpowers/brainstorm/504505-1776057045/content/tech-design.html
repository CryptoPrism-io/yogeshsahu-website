1 — Veo 3 Scroll Animation (Hero Centrepiece)

How it works

Generate a ~6–10 second cinematic video with Veo 3 / Google Flow. Extract ~180 frames at 30fps using ffmpeg. Load frames as a WebP sprite sequence. Drive playback via the user's scroll position using the Canvas API + IntersectionObserver.

Performance strategy
  • Frames preloaded in batches of 20 on idle (requestIdleCallback)
  • WebP format — ~60% smaller than JPEG
  • Canvas renders at 0.5× DPR on mobile
  • Fallback: static poster image if frames fail to load
  • Total frame payload: ~5–6MB (acceptable for hero)
ffmpeg extraction command
ffmpeg -i avatar.mp4 \
  -vf "fps=30,scale=1080:-1" \
  -q:v 85 \
  public/frames/frame_%04d.webp

Run once, commit frames to public/frames/. Next.js serves them statically.

4 — Animation Inventory

1

Hero scroll animation — Canvas API

Veo 3 frames driven by scroll position. Sticky hero, 500vh scroll zone. Smoothed with linear interpolation to prevent jitter.

2

Section reveal — Framer Motion

Each section fades up (y: 40 → 0, opacity: 0 → 1) as it enters the viewport. 0.6s ease. Triggered once per section.

3

Stat counters — Framer Motion animate

2.26M, 23, 50K, 21 count up from 0 when About section enters viewport. Duration: 1.5s ease-out.

4

Marquee — CSS animation

Pure CSS @keyframes translateX loop. No JS dependency. Pauses on hover.

5

Project card hover — Framer Motion whileHover

Gold border appears, card lifts 4px, subtle gold glow. 200ms ease.

6

Nav transition — CSS

Transparent on hero, transitions to rgba(8,8,8,0.92) + blur(12px) on scroll past hero. Pure CSS transition.
